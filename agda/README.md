# Formally verifying `batchSimultaneous`

The goal of this directory: a machine-checked proof that `batchSimultaneous`
behaves correctly **for any combination of the primitives** (`of`, `empty`,
`map`, `take`, `merge`, and eventually the higher-order joins). The TypeScript
test suite and the fast-check oracle sample that space; the Agda development is
where we close it.

The whole approach rests on one move: replace observables — IO-bound,
push-based, order-sensitive — with a pure *denotation* that carries the same
information. Then correctness statements become equalities between list
programs, provable without ever modeling subscriptions, schedulers, or time.

## Idea 1 — observables as monotonic timed lists

An observable *is* its history: model `Observable<A>` as a list of
`(timestamp, value)` pairs with monotonically non-decreasing timestamps, and an
`Instantaneous<A>` as such a list tagged with a provenance. The combinators
become list programs that are transparently about **preserving monotonicity**:

- **merge** — sort-merge the two lists by timestamp. Stability (on equal
  timestamps, the left argument's emissions come first) is the denotational
  counterpart of rxjs subscription order.
- **concat** — keep the first list; shift the second list's timestamps past
  the end of the first, then append.
- **switch** — like merge, except the first list's emissions stop once the
  second has started.
- **map / take** — pointwise on values / a prefix.

"Simultaneous" is now simply **timestamp equality**, and `batchSimultaneous`
has an obvious specification: group equal timestamps. The problem that is
subtle in push-land — you can't know an emission is the first of a group, or
that another is the last, without time travel — is trivial in the denotation,
because the whole history is a value. This is what lets us make assertions
about IO-based entities without IO.

One constraint must be engineered in: *instantaneouses that don't share a
provenance must never be simultaneous*. The original scheme: with N root
sources, source `p` may only emit at timestamps `≡ p (mod N)` — each source
picks its timestamps from the arithmetic sequence generated by its provenance,
so distinct sources can never collide.

The development uses a structural refinement of the same idea. Raw naturals
have a flaw: any operator that does timestamp arithmetic (concat's shift!)
moves timestamps into arbitrary residue classes, silently breaking the
invariant — it would have to be re-proved after every operation. So instead,
a `Time` is a **pair**:

```agda
Time = ℕ × ℕ            -- (tick , origin)
```

ordered lexicographically — which is exactly the order induced by the
injection `(tick, origin) ↦ tick * N + origin`, i.e. the arithmetic-sequence
scheme, promoted from a convention you must maintain to a definition that
cannot be violated. "Distinct origins are never simultaneous" is now the
injectivity of a pairing, proved once ([src/Time.agda](src/Time.agda)); a
shift only touches the `tick` component, so the invariant survives every
operator *by construction*, and no other proof ever touches modular
arithmetic.

## Idea 2 — programs as expression trees (the deep embedding)

"…given **any combination** of the primitives" is a quantification over
*programs*. To state it inside Agda at all, programs must be data. So the
primitives are reified as an inductive syntax:

```agda
data Exp : Set where
  srcE   : ℕ → Exp                 -- a root source, by index
  emptyE : Exp
  mapE   : (Val → Val) → Exp → Exp
  takeE  : ℕ → Exp → Exp
  mergeE : Exp → Exp → Exp
```

together with a denotation `⟦_⟧ : Exp → Env → TimedObs Val`, where an
environment assigns every root source its timed list
([src/Exp.agda](src/Exp.agda)). Two sub-expressions referring to the same
`srcE i` — that is literally the diamond.

Now the target theorem is expressible: **∀ (e : Exp) → …**, and the induction
we were looking for has a name — *structural induction on the syntax*. One
case per primitive; each case is a lemma about timed lists. There is no other
way to get "for any combination" into a proof assistant.

The embedding earns its keep twice over: the TypeScript oracle
([typescript/src/model/exp.ts](../typescript/src/model/exp.ts)) uses the *same*
`Exp` type to generate thousands of random programs and check the real rxjs
implementation against the timed-list model. The oracle is the fast, empirical
approximation of the theorem; the Agda development is its limit. Anything the
model gets wrong is found by fast-check in seconds, long before it can be
fossilized into a proof.

## The proofs themselves

### Architecture: specification vs implementation

`batchSimultaneous` exists at two levels, deliberately kept apart:

1. **The specification** — in the denotation, batching is just *group
   consecutive equal Times* (`batchSpec` in
   [src/TimedObs.agda](src/TimedObs.agda)). Equal times are adjacent in any
   merged list, so this is a five-line fold. It is obviously the right
   semantics, and that obviousness is the point: laws are cheap to state and
   prove against it.
2. **The implementation** — the actual algorithm from
   [typescript/src/v5/batch-simultaneous.ts](../typescript/src/v5/batch-simultaneous.ts)
   (emission trees, the provenance memory, `awaitingValueCount`/`totalNum`
   windows), to be ported as a pure function over an ordered replay of the
   timed list.

The endgame theorem relates them, by induction on the syntax:

```
∀ (e : Exp) (env : Env) → ⟦ batchSimultaneousImpl e ⟧ env ≡ batchSpec (⟦ e ⟧ env)
```

Each primitive's case will state how the memory corresponds to the denotation
— a simulation invariant of the shape *"`totalNum p` equals the number of live
subscriptions of provenance `p` in the surrounding context"*. This is exactly
the invariant the implementation bugs found on 2026-07-05 violated (the
lone-source off-by-one, the wrapper-gating drop, the switchMap unit-init
problem): each of them would have surfaced here as an unprovable case.

### What is proved today

Three theorem groups, each a different kind of "for any combination":

1. **`deep-diamond`** ([src/Deep.agda](src/Deep.agda)) — *the arbitrary-depth
   theorem.* For **any** expression built from `srcE i` / `mapE` / `mergeE`,
   nested to any depth (a merge of a merge of a merge of …, with maps
   anywhere):

   ```agda
   deep-diamond : DiamondOver i e → StrictMono (emits (env i))
     → batchSpec (emits ⟦ e ⟧) ≡ mapT (applyAll (funs e)) (emits (env i))
   ```

   Batching the denotation yields exactly **one batch per source instant**,
   containing the value of every root-to-leaf path (`funs e` = the composed
   map-functions along each path) in left-to-right order. The proof is
   structural induction on the expression, through a normal form: every such
   tree denotes `expand (funs e) source` (each source emission becomes a
   block of k simultaneous emissions), established by block-pulling lemmas
   about stable merge. The two-way diamond is the depth-one instance
   (`deep-diamond-two`, with `funs = [id, id]`), and the implementation port
   satisfies the same law (`impl-deep-diamond`).

2. **`denote-wf`** ([src/Exp.agda](src/Exp.agda)) — any combination of ALL
   the primitives (`src`, `empty`, `of`, `map`, `take`, `merge`, `concat`),
   at any depth, denotes a well-formed observable: time-ordered emissions,
   all at or before the close. One induction case per primitive; the weight
   lives in per-operator preservation theorems
   ([src/Obs.agda](src/Obs.agda), [src/Sorting.agda](src/Sorting.agda)) —
   the machine-checked version of "these are all just approaches to
   preserving monotonicity".

3. **`batchImpl-spec`** ([src/BatchImpl.agda](src/BatchImpl.agda)) — the
   correspondence theorem: the implementation port (the accumulate-and-flush
   fold that is the observable behavior of the TS scan) agrees with the
   specification on **every** stream whatsoever:

   ```agda
   batchImpl-spec : (xs : TimedObs A) → batchImpl xs ≡ batchSpec xs
   ```

   Since it holds for all streams, it holds for the denotation of every
   expression — no side conditions. Proved via a `joinHead` bridge relating
   the fold's accumulator to the spec's right-fold grouping.

4. **`batchCount-impl`** ([src/BatchCount.agda](src/BatchCount.agda)) — the
   *counting mechanism*. `batchImpl` still flushes by comparing timestamps,
   which the TS code cannot do; `batchCount` is the real mechanism: `mult`
   is `totalNum`, a countdown is `awaitingValueCount`, the accumulator is
   the memory's `batch`, and **no time comparison decides a flush**. The
   simulation theorem says truthful counting equals time-comparison
   flushing:

   ```agda
   batchCount-impl : Instants mult xs → batchCount mult xs ≡ batchImpl xs
   ```

   `Instants mult xs` is the simulation invariant, formalized: *the recorded
   multiplicity of every instant equals the number of deliveries it makes* —
   "totalNum = live subscriptions". `expand-instants` discharges it for the
   whole diamond fragment (a merge tree with k leaves delivers every source
   instant exactly k times), which closes the tower end to end
   (`count-deep-diamond`):

   ```
   counting mechanism ≡ flush behavior ≡ specification ≡ one batch per instant
        (batchCount)      (batchImpl)     (batchSpec)      (deep-diamond)
   ```

   Modeling honesty: the memory here holds one open instant rather than a
   map keyed by provenance (justified on sorted streams, where an instant's
   deliveries are adjacent, so at most one window is ever open), and `mult`
   is static — matching the fragment where no branch closes early.

5. **`indep-diamonds`** ([src/Indep.agda](src/Indep.agda)) — *independent
   diamonds don't interfere.* The origin discipline earning its keep:
   streams with distinct origins can never share an instant, so batching
   distributes over their merge —

   ```agda
   batch-merge-indep : eqℕ i j ≡ false → OriginIs i xs → OriginIs j ys
     → batchSpec (mergeT xs ys) ≡ mergeT (batchSpec xs) (batchSpec ys)
   ```

   — and therefore a merge of two arbitrary-depth diamonds over *different*
   sources batches each diamond independently (`indep-diamonds`): the
   generalization of the TS test "keeps two independent sources in separate
   batches". The proof pivots on two head-level facts: inserting a value
   strictly before everything in the other stream commutes with the merge
   (`insert-merge-left/right`).

6. **`take-diamond` / `count-take-diamond`**
   ([src/TakeDiamond.agda](src/TakeDiamond.agda)) — *a branch that closes
   early, and dynamic multiplicities.* For `merge(a, take n a)`:

   ```agda
   take-diamond : StrictMono xs
     → batchSpec (mergeT xs (takeT n xs)) ≡ takeDiamondSpec n xs
   ```

   — the first n instants batch doubled, every later instant batches alone
   (the TS `[[1,1],[2]]` test, as a law). And the counting mechanism
   satisfies it with a **dynamic multiplicity**: `takeMult n xs` reports 2
   on taken instants and 1 afterwards — `totalNum` dropping when a branch
   closes — proved truthful (`take-instants`, via an `Instants`-transport
   lemma showing the invariant only reads multiplicities at the stream's
   own instants) and pushed through the whole tower (`count-take-diamond`).

Close times ([src/Obs.agda](src/Obs.agda)) make `take` and `concat`
definable (`takeClose`, `filterAfter` past the pivot), and the `of` origin
discipline is formalized: all values of one `of` occupy one instant
(`of-batch`), and distinct origins can never be simultaneous
(`distinct-origins` in [src/Time.agda](src/Time.agda)).

### The first proof: the anchor law

The first law worth having is the diamond itself:

```
batchSimultaneous (merge a a) ≡ map (λ x → [ x , x ]) a
```

In [src/Diamond.agda](src/Diamond.agda) this is:

```agda
diamond : (xs : TimedObs A) → StrictMono xs
        → batchSpec (mergeT xs xs) ≡ mapT dbl xs
```

for any stream with strictly increasing times — the shape of every root
source's denotation (one emission per instant). It lifts to the syntax as
`diamond-exp` in [src/Exp.agda](src/Exp.agda): for *any* expression whose
denotation is strictly monotone, batching its self-merge doubles every value.

The proof decomposes the way the diamond actually happens:

1. **`merge-dup-tail`** — if `x` precedes everything in `xs`, then
   `mergeT xs (x ∷ xs) ≡ x ∷ mergeT xs xs`. One comparison gets rewritten
   (`timeLt x y → timeLeq y x ≡ false`) and the goal reduces.
2. **`merge-dup`** — therefore a self-merge duplicates its head twice over:
   `mergeT (x ∷ xs) (x ∷ xs) ≡ x ∷ x ∷ mergeT xs xs`. This is where
   *stability* does its work: on the tie, the left copy goes first.
3. **`diamond`** — induction on `xs`. The head's two copies are adjacent, the
   strict-monotonicity hypothesis says the next group has a different time, so
   `batchSpec` closes the pair into `[v, v]` and the induction hypothesis
   handles the rest.

### Design choices that keep the proofs small

These are worth knowing before extending the development — they are the
difference between three-line cases and hundred-line ones:

- **Self-contained prelude** ([src/Prelude.agda](src/Prelude.agda)) — no
  standard library. Booleans, naturals, lists, pairs, `_≡_`, and precisely the
  comparison lemmas the order needs (`eqℕ-sound`, `ltℕ-asym`,
  `ltℕ⇒leqℕ-flip-false`, …). Nothing to install, nothing to version-match.
- **Boolean comparisons + soundness lemmas**, not decidable-order records.
  `mergeT` and `insertBatch` branch on `if timeLeq … / if timeEq …`; every
  proof is then a chain of `rewrite`s that flips the relevant comparison to a
  literal `true`/`false`, after which the goal *computes*. Almost every proof
  in [src/Diamond.agda](src/Diamond.agda) ends in `refl`.
- **`if_then_else_` instead of `with`** in the program definitions, so the
  functions stay ordinary terms that `rewrite` can drive. (`with`-abstraction
  is used only inside the order lemmas themselves, where it is local.)
- **`batchSpec` as a right fold** (`insertBatch` per element) — no
  accumulator, so induction on the list is induction on the spec, and the
  helper lemma "an element joins the first group of the tail iff the times
  match" is definitional.
- **`StrictMono` as an inductive predicate** mirroring list structure, so the
  hypothesis destructs in lockstep with the list in every induction.

### Roadmap

Done (2026-07-05):

1. ~~**Close times**~~ — `record Obs A = { emits ; close }` with
   per-operator well-formedness preservation, `takeClose`, and hot-source
   `concat` via `filterAfter`. (The *cold* case — `of` under `concat`, whose
   emissions move to the subscription instant — needs subscription-time-
   parameterized denotations; still open.)
2. ~~**`of` with an origin discipline**~~ — `ofE origin values`, one instant
   per occurrence, `of-batch` + `distinct-origins`.
3. ~~**Monotonicity-preservation laws**~~ — merge/map/take/filterAfter/append
   preserve sortedness and boundedness ([src/Sorting.agda](src/Sorting.agda)),
   assembled into `denote-wf`.
4. ~~**The buffering port + correspondence**~~ — `batchImpl-spec`, and the
   arbitrary-depth `deep-diamond`.

5. ~~**The count-based memory refinement**~~ — `batchCount` + the `Instants`
   invariant + the simulation theorem `batchCount-impl`, discharged for the
   diamond fragment by `expand-instants` and closed end-to-end by
   `count-deep-diamond` ([src/BatchCount.agda](src/BatchCount.agda)).

6. ~~**Dynamic multiplicities**~~ — for the canonical branch-closure scenario:
   `takeMult` varies per instant (2 → 1 when the take closes), proved
   truthful and pushed through the tower
   ([src/TakeDiamond.agda](src/TakeDiamond.agda)). Because instants are
   distinct Times, per-Time multiplicity covers it; a keyed memory only
   becomes observable once interleaved *open windows* exist, which sorted
   single-instant streams rule out.
7. ~~**Multi-source trees**~~ — `batch-merge-indep` + `indep-diamonds`
   ([src/Indep.agda](src/Indep.agda)): batching distributes over
   origin-disjoint merges, so independent diamonds compose.

8. ~~**The joins: the semantic decision**~~ — **DECIDED (option 1,
   non-transitive causal batching)**: an inner spawned by an *async* trigger
   inherits the trigger's instant exactly (its values are simultaneous with
   the event that caused them); an inner subscribed in a *synchronous burst*
   is a fresh root cause. Encoded in the TypeScript oracle
   ([typescript/src/model/exp.ts](../typescript/src/model/exp.ts),
   `mergeMap` node) and validated against the implementation on random
   trees. The async half is proved as a law
   ([src/MergeMap.agda](src/MergeMap.agda)): `bindT` is the time-preserving
   bind, and

   ```agda
   mergeMap-diamond : StrictMono xs
     → batchSpec (mergeT xs (bindT g gs xs)) ≡ mapT (λ v → v ∷ g v ∷ gs v) xs
   ```

   — `merge(a, a.mergeMap(inner))` batches `[v, …inner v]` per instant
   (`impl-mergeMap-diamond` for the port). The sync half needs no new
   theorem: a sync-triggered inner is semantically a fresh `of`, governed by
   `of-batch` and `batch-merge-indep`.

9. ~~**Nested joins**~~ — implemented (`unitDelivery` in
   [typescript/src/v5/joins.ts](../typescript/src/v5/joins.ts)): everything
   one async delivery produces synchronously — sibling inners' values,
   registrations, closes — coalesces into ONE unit anchored at the
   provenance that owns the delivered instant (`anchorProvenance` follows
   the async chain), and the unit's subtree gets population-only memory
   bookkeeping (`registerSubtree`). Nested `mergeMap`s, diamonds as
   triggers, and their combinations now batch compositionally
   (`[5, 105]`, `[5, 5, 105]`, `[105, 105]` — pinned in the oracle suite),
   and the random sweep runs with a fully unrestricted generator. The sweep
   also caught and fixed a latent window-release bug (a sibling closing
   mid-window dropped `totalNum` to 1 and the no-diamond shortcut stranded
   the buffered batch — the shortcut now requires no window in flight).

10. ~~**Take in the deep embedding**~~ —
    ([src/TakeDeep.agda](src/TakeDeep.agda)) expansion with per-leaf
    horizons: each root-to-leaf path carries its composed function plus a
    `Horizon` (takes cap it, `capH`), leaves die as horizons expire, and

    ```agda
    dt-diamond : DTOver i e → StrictMono (emits (env i))
      → batchSpec (emits ⟦ e ⟧) ≡ specH (funsH e) (emits (env i))
    ```

    — any merge/map tree with takes on flat branches, at any depth, batches
    to one batch per source instant holding every LIVE leaf's value, and no
    batch once all horizons expire. `dt-take-two` recovers TakeDiamond's
    `[[1,1],[2]]` law as the two-leaf instance, tying the modules together.
11. ~~**Nested binds**~~ — `bind-fusion` in
    [src/MergeMap.agda](src/MergeMap.agda): a join over a join at inherited
    times is one join with the composed inner
    (`bindT g′ gs′ ∘ bindT g gs ≡ bindT (fused)`), the Agda counterpart of
    the `unitDelivery` semantics; `nested-mergeMap-diamond` gives the
    batched law for `merge(a, a.mergeMap(f).mergeMap(f′))`.

12. **`share` — the aliasing primitive** ([src/Share.agda](src/Share.agda)).
    share preserves the upstream provenance across subscribers, which is
    what justifies every theorem that reuses one timed list on both sides
    of a merge — the diamond theorems were implicitly share theorems all
    along. The new law is the LATE subscriber (registration-only replay:
    it missed the first n instants), the time-reversed dual of take:

    ```agda
    late-diamond : StrictMono xs
      → batchSpec (mergeT xs (dropT n xs)) ≡ lateDiamondSpec n xs
    ```

    — single batches before the subscriber joins, doubled after
    (`[[1],[2,2]]`, mirroring take's `[[1,1],[2]]`), with the counting
    version (`count-late-diamond`) whose dynamic multiplicity RISES 1 → 2
    as the subscriber joins (`lateMult`, mirroring `takeMult`'s 2 → 1).
    The TS implementation is refcounted with registration-only replay, and
    the oracle's `letShare`/`shareRef` embedding validates shared derived
    pipelines on random programs. One recorded approximation: intra-batch
    order for shared fan-out follows the implementation's delivery tree
    (a share's refs receive consecutively at its connection point) while
    the model idealizes flat pre-order — the oracle compares
    share-containing programs up to intra-batch order, exact fan-out order
    is pinned by unit tests, and a delivery-path order model is the future
    refinement.

13. **The two-sorted embedding: the join as the primitive**
    ([src/Derived.agda](src/Derived.agda)). The canonical primitive set
    makes `mergeAll` primitive; `merge` and `mergeMap` are now **theorems**:

    ```agda
    merge-derived    : mergeAllD (ofS (a ∷ b ∷ [])) env ≡ mergeT ⟦a⟧ ⟦b⟧
    mergeMap-derived : AsyncOnly ⟦e⟧
                     → mergeAllD (mapS g gs e) env ≡ bindT g gs ⟦e⟧
    ```

    The second sort `ExpS` (`ofS` — a sync burst of inners; `mapS` — inners
    from triggers) gets its own denotation, with `bindTSplit` carrying the
    origin supply for sync-burst triggers (the oracle's rule, verbatim);
    on async triggers it collapses to `bindT`. The batch-shape theorems
    transfer to the primitive forms by composition (`mergeAll-of-diamond`,
    `mergeAll-map-diamond`). Oracle-side, the generator now produces
    `mergeAll` directly over both shapes (n-ary bursts included), and the
    derivation laws are checked as program equivalences — model and
    implementation — on random subexpressions.

14. **Dynamic inner arrival — subscribers that join over time**
    ([src/Grow.agda](src/Grow.agda)). The full-`mergeAll` frontier: a hot
    shared stream subscribed afresh by *each* trigger of a join
    (`x.pipe(map(v => hot), mergeAll())`) accumulates late subscriptions at
    the triggers' join points. This is [src/TakeDeep.agda](src/TakeDeep.agda)
    time-reversed — leaves carry **delays** instead of horizons (born later
    instead of dying early). The normal form `expandD` emits, per instant,
    the values of the already-born leaves, and the theorem quantifies over
    ANY family of join points:

    ```agda
    grow-diamond : StrictMono xs
      → batchSpec (mergeList (xs ∷ map (λ n → dropT n xs) ns))
      ≡ specD ((id , 0) ∷ map (λ n → (id , n)) ns) xs
    ```

    — one batch per instant whose multiplicity GROWS as subscribers arrive
    (`[[7],[8,8]]` for one join at instant 1), with `specD-late` recovering
    Share's `lateDiamondSpec` as the single-subscriber instance. Nearly
    every lemma is reused across time's arrow: `merge-expandD` and
    `batch-expandD` are `merge-expandH`/`batch-expandH` with `pull-left-vals`,
    `pull-right-vals`, `batch-cons-block` and the `AllAfter` machinery
    unchanged. Oracle-side, `mapShareS` (hot inners under `mergeAll∘map`)
    denotes each trigger as the shared list's strict suffix and folds the
    suffixes with `mergeTimed`; the random sweep exercises it alongside all
    other shapes. Chasing this law flushed out two real implementation
    bugs: a routing-level decrement in `batchSimultaneous` that corrupted
    windows when a source provenance doubled as a join anchor, and
    `flipInside` re-tagging unit-init trees with the wrapper provenance,
    destroying the instant-owner anchor. One recorded grammar frontier:
    self-triggering hot joins (`x.pipe(map(v => x), mergeAll())`, the shared
    stream triggering its own re-subscription) are excluded from the
    generator — the re-subscription's registration is indistinguishable
    from an empty-inner marker in the current emission protocol.

15. **The serial joins: close-time threading** (oracle-side; Agda pending).
    `concatAll`, `switchAll` and `exhaustAll` mirror rxjs (checked by
    side-by-side comparison tests running the same program through plain
    rxjs), and their denotations thread CLOSE TIMES: the model returns
    `{list, close}` where `close` is `"frame"` (completes within its own
    subscription instant) or the tick of the async instant it completes at
    — sources close at drive end, `take` at its nth value, joins by
    max/threading. concatAll subscribes each inner at the previous close;
    switchAll passes every burst inner's subscription-frame emissions and
    keeps only the last live; exhaustAll walks the burst, blocked by the
    first inner with a pending close (the rest are never subscribed). The
    semantic decision, consistent with option 1: **a mid-stream
    subscription instant behaves exactly like the root one** — a queued
    cold's values are one fresh instant at the closing tick (ordered right
    after their cause, never batched with it), and its sync-burst mergeMap
    inners are fresh root causes. The bind rule is now keyed on the
    ORIGIN CLASS (cold-born vs driver instants), not on tick 0.
    Chasing this law flushed out two model bugs (closes pointing into the
    past for already-completed sources; `"frame"` closes reported relative
    to the wrong frame after the queue advanced) and two implementation
    bugs, both the same protocol ambiguity: a mid-stream top-level init
    (a late subscription instant) was converted into an async-wrapped
    UNIT by enclosing join wrappers — `flipInside` now keeps mid-stream
    init batches init-headed, and every join's async branch routes
    init-headed emissions through the sync-burst path (only async-headed
    emissions are value-triggered unit deliveries). The unsubscribe-close
    protocol gap is CLOSED: `switchWithCloses` tracks each inner's
    registration balance (`registrationDeltas`, mirroring the memory's
    totalNum bookkeeping) and synthesizes decrement-only closes
    (`async{p, close}` — an `init{p,[close]}` would register then close,
    net zero) when a live inner is switched away, with `groupSync`
    preserving async-headed closes as pure decrements. Diamonds across a
    switch batch correctly and the generator allows arbitrary live
    switched-away inners.

The primitive set is Anthony's canonical list — `of`, `empty`, `map`,
`take`, `accumulate`, `share`, `mergeAll`, `concatAll`, `switchAll`,
`exhaustAll` — with `merge`/`mergeMap` derived. Next frontiers: Agda
denotations for the serial joins (close-time threading in the deep
embedding); shares under serial joins (late share-connection modeling);
async stream-of-streams outers for the serial joins; mid-instant take
cuts; delivery-path intra-batch order; `accumulate` in the model (a
stateful fold over values in time order — provenance-transparent,
batching-wise identical to map); self-triggering hot joins (needs a
distinguished re-subscription marker in the emission protocol);
take-over-merge counting.

## Building

```sh
agda src/Everything.agda
```

registration needed — the development is self-contained.
